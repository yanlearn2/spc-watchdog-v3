## 三、数据流

### 3.1 一次扫描周期

```
打开Excel → 遍历工作表 → 白名单判断
                          ├─ 不匹配 → 跳过
                          └─ 匹配 → 提取标准行规格
                                    ├─ 写入 qc_specs 表
                                    └─ 逐行扫描数据
                                       ├─ 白名单过滤
                                       ├─ 行指纹去重(MD5)
                                       ├─ 写入 samples 表
                                       ├─ 检查超限
                                       │   ├─ 自动规格→报警检查
                                       │   ├─ spec_overrides→覆盖阈值
                                       │   └─ mode判断(alarm/monitor)
                                       └─ 超限→写入 alarm_log 表
                               → 汇总报警 → 按样品分组
                                         → 发飞书卡片消息
                                         → 写 CSV 备份
```

### 3.2 数据库接口架构（可插拔）

```
DatabaseBackend（抽象类）
    ├── SQLiteBackend（当前实现）
    ├── MySQLBackend（预留）
    ├── PostgreSQLBackend（预留）
    └── FeishuBitableBackend（预留）

添加新数据库只需：
  ① 继承 DatabaseBackend
  ② 实现所有抽象方法
  ③ 在 create_backend() 中注册
```

## 四、核心设计决策

### 4.1 白名单机制

- **目的**：全厂17个车间、5万+行数据，用户只关心10~20个样品类型
- **实现**：三级过滤
  1. 工作表级 → 检查标准行是否匹配白名单
  2. 标准行级 → 只提取匹配白名单的规格
  3. 数据行级 → 只处理匹配白名单的样品
- **配置**：`config.json → filter.whitelist_patterns`

### 4.2 规格自动提取

- 从 Excel 标准行（R2~R6）自动解析规格值
- 支持格式：`≤10g/L` `≥80` `80~110` `0.005~0.015(参考)`
- 自动识别 `(参考)` → 降级为 monitor 模式
- 跳过 `实测` `NotNull` `IsNull` 等非量化规格

### 4.3 规格覆盖（Spec Overrides）

- **不修改Excel**（防止被还原）
- 通过 config.json 的 `spec_overrides` 单独调整报警阈值
- 原始规格保留显示，仅报警判断用覆盖值
- 格式：`"样品正则/项目": {"alarm_upper": 0.6}`

### 4.4 报警分级

| 模式 | 行为 | 触发条件 |
|------|------|---------|
| `alarm` | 发飞书通知 + 写数据库 | 无 `(参考)` 标记 |
| `monitor` | 仅写日志+数据库 | 规格带 `(参考)` 标记 |
| 覆盖 | 按 config.json 强制指定 | `alarm_mode.overrides` |

### 4.5 飞书通知分组

- 按样品名称分组展示
- 每组显示样品名、检测项、检测值、规格、偏差
- 规格覆盖时注明 `（报警≥X）`
- 超长内容自动分段发送

## 六、部署步骤

### 6.1 环境要求
- Python 3.8+
- 依赖安装：`pip install -r requirements.txt`

### 6.2 配置
1. 编辑 `config.json`：
   - **单文件模式**：修改 `excel.path` 为实际路径
   - **多文件模式**：在 `excel.files` 数组中添加文件条目（每个文件可独立配置密码和排除表）
   - 设置 `feishu.webhook_url`（已预填）
   - 按需调整 `filter.whitelist_patterns` 和 `spec_overrides`

### 6.3 首次部署
```bash
# 1. 安装依赖
pip install -r requirements.txt

# 2. 初始化数据库
python watchdog.py --init-db

# 3. 测试扫描
python watchdog.py --once

# 4. 持续监控
python watchdog.py
```

### 6.4 Windows部署
```batch
# 方式一：首次部署脚本
双击 deploy.bat

# 方式二：持续运行
双击 run_watchdog.bat

# 方式三：直接命令
python watchdog.py --once     # 测试一次
python watchdog.py             # 持续监控（Ctrl+C停止）
```

### 6.5 稳定性特性
- **文件锁定重试**: Excel被打开时自动重试（指数退避，最多3次）
- **优雅退出**: 按 Ctrl+C 信号处理后完成当前周期再退出
- **DB健康检查**: 每10个周期自动检查数据库连接，断线自动重连
- **日志滚动**: 日志文件自动滚动（默认50MB/个，保留3个备份）

## 七、数据库表结构

### samples 表 — 原始检测数据
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| row_hash | TEXT UNIQUE | 行指纹(MD5去重) |
| sample_name | TEXT | 样品名称 |
| test_date | TEXT | 检测日期 |
| workshop | TEXT | 车间 |
| source_file | TEXT | 来源文件名 |
| data_json | TEXT | 全部检测结果JSON |
| synced_at | DATETIME | 同步时间 |

### alarm_log 表 — 报警记录
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| sample_name | TEXT | 样品名称 |
| item | TEXT | 检测项目 |
| value | REAL | 检测值 |
| spec | TEXT | 规格描述 |
| deviation | TEXT | 偏差百分比 |
| mode | TEXT | alarm/monitor |
| level | TEXT | 一级/二级/三级 |
| status | TEXT | 待处置/已处置/已闭环 |
| alarmed_at | DATETIME | 报警时间 |

### qc_specs 表 — QC规格
| 字段 | 类型 | 说明 |
|------|------|------|
| id | INTEGER | 主键 |
| sample_pattern | TEXT | 样品名称正则 |
| item | TEXT | 检测项目 |
| upper_limit | REAL | 规格上限 |
| lower_limit | REAL | 规格下限 |
| mode | TEXT | alarm/monitor |
| source | TEXT | auto/manual |

## 八、已知限制
1. 目前只支持 SQLite（预留了 MySQL/PostgreSQL 接口但未实现）
2. Excel 加密需 `msoffcrypto-tool` 库
3. 飞书WebHook不支持富文本表格，使用缩进分组替代
4. 连续100个空行后停止扫描（防无限空行）
